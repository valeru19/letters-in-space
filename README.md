# Лабораторная работа №3: Основные представления трёхмерных объектов

Проект реализует интерактивное 3D-приложение на `PySide6` без использования сторонних 3D-движков и специальных модулей PySide для готовой 3D-графики.

В сцене используются две объёмные буквы русского алфавита: **`М`** и **`П`**.

---

## 1. Возможности программы

### Этап 1. Каркасная модель
- Построены 3D-модели букв `М` и `П` (полигональные меши).
- Для каждой буквы отдельно задаются:
  - ширина,
  - высота,
  - глубина.
- Реализованы преобразования объекта:
  - смещение,
  - вращение,
  - отражение по осям `X/Y/Z`,
  - приближение/удаление.
- Реализованы преобразования камеры:
  - смещение,
  - вращение.
- Есть автомасштабирование (`Автомасштаб`) для удобного обзора сцены.
- Используется перспективная проекция (мировые координаты -> экранные).

### Этап 2. Видимые части
- Реализован метод скрытия невидимых поверхностей: **Z-буфер**.
- Корректная отрисовка при аффинных преобразованиях.
- Независимое управление каждой буквой.
- Поддерживается пересечение букв.

### Этап 3. Закрашивание поверхностей
- Поддерживаемые режимы:
  - `Монотонная (Flat)`,
  - `Гуро`,
  - `Фонг`.
- Можно менять положение источника света (`X/Y/Z`).
- Источник света отображается в окне сцены.

## 4. Интерфейс программы

Окно состоит из двух частей:

- **Слева**: холст отрисовки 3D-сцены.
- **Справа**: панель управления:
  - выбор активной буквы (`М`/`П`);
  - параметры размеров;
  - выбор цвета активной буквы;
  - шаги смещения/вращения;
  - кнопки преобразований объекта;
  - кнопки управления камерой;
  - выбор режима рендера;
  - опции отладки (нормали, локальные оси объектов);
  - параметры источника света;
  - сохранение/загрузка сцены в `json`;
  - текстовая подсказка по горячим клавишам и мыши.

---

## 5. Управление

## 5.1 Клавиатура

- `Стрелки` — смещение выбранной буквы по `X/Y`.
- `W/S` — смещение выбранной буквы по `Z`.
- `Q/E` — вращение выбранной буквы вокруг `Y`.
- `A/D` — вращение выбранной буквы вокруг `X`.
- `Z/C` — вращение выбранной буквы вокруг `Z`.
- `Shift` + те же клавиши — управление **камерой** вместо буквы.

## 5.2 Мышь на холсте

- `ЛКМ` — вращение выбранной буквы.
- `Ctrl + ЛКМ` — перемещение выбранной буквы в плоскости экрана.
- `ПКМ` или `Shift + ЛКМ` — орбитальное вращение камеры (Blender-подобно).
- `Средняя кнопка мыши` — панорамирование камеры.
- `Колесо` — приближение/удаление выбранной буквы.
- `Shift + Колесо` — приближение/удаление камеры.

---

## 6. Режимы рендера

Переключаются в блоке `Рендер`:

- `Каркас` — отрисовка рёбер.
- `Видимые части (Z-буфер)` — только видимые поверхности.
- `Монотонная (Flat)` — закраска грани одним тоном.
- `Гуро` — интерполяция яркости по вершинам.
- `Фонг` — интерполяция нормалей и расчёт освещения по пикселям.

Дополнительно доступны:
- `Рёбра поверх заливки`;
- `Показывать нормали` (визуализация нормалей граней);
- `Показывать локальные оси объектов` (локальные XYZ для каждой буквы).

## 7. Структура проекта

- `main.py`  
  Точка входа: запуск `QApplication` и создание окна.

- `math3d.py`  
  Математика, данные сцены и генерация геометрии:
  - `Vec3`, `clamp`, `apply_euler`, `apply_inverse_euler`,
  - `Mesh`, `Object3D`, `Camera`,
  - `build_letter_em()`, `build_letter_pe()`, `build_default_objects()`.

- `renderer.py`  
  Класс `RenderWidget`:
  - отрисовка каркаса и закраски,
  - Z-буфер,
  - режимы Flat/Gouraud/Phong,
  - отображение света, нормалей и локальных осей,
  - интерактивное управление мышью в стиле Blender.

- `ui.py`  
  Класс `MainWindow`:
  - сборка интерфейса,
  - обработчики кнопок/клавиш,
  - выбор цвета объекта,
  - сохранение и загрузка сцены (`json`).

## 8. Как устроен код (основные сущности)

Ниже краткая карта классов и функций:

- `Vec3`  
  Базовая векторная математика: сложение, вычитание, нормализация, скалярное/векторное произведение.

- `Mesh`  
  Хранит геометрию объекта:
  - `vertices` (вершины),
  - `faces` (треугольники),
  - `edges` (рёбра для каркаса).

- `Object3D`  
  Описывает конкретный объект сцены (букву):
  - размеры, позицию, поворот, отражение,
  - метод `world_vertices()` вычисляет вершины в мировых координатах.

- `Camera`  
  Положение, углы, `fov`, `near`.  
  Метод `world_to_camera()` переводит мировые координаты в систему камеры.

- `build_letter_em()`, `build_letter_pe()`  
  Формируют меши букв `М` и `П` из набора прямоугольных блоков.

- `RenderWidget`  
  Основной рендерер и интерактивный холст:
  - проекция (`project()`),
  - каркасная отрисовка,
  - raster + Z-буфер,
  - режимы закраски Flat/Gouraud/Phong,
  - работа с источником света,
  - управление мышью (объект/камера).

- `MainWindow`  
  Создаёт интерфейс, связывает элементы управления, а также отвечает за сохранение/загрузку сцены.

## 9. Важные математические детали

- Преобразования объекта: `масштаб -> отражение -> вращение -> перенос`.
- Для камеры используется обратное преобразование (из мира в вид камеры).
- Перспективная проекция:
  - чем больше `z` (глубина), тем меньше объект на экране.
- Для скрытия поверхностей используется Z-буфер:
  - для каждого пикселя хранится ближайшая глубина,
  - дальние фрагменты отбрасываются.
